Java 秒杀演进实验室

这个页面现在走真实 MySQL 数据库。你可以看见 select、update、insert 这些数据库动作是怎么一步步导致超卖、又如何通过条件更新和用户锁解决问题的。

连接方式

如果你是从 IDEA 的 HTML 预览页打开本页面，默认的相对路径会请求到 IDEA 自己的内置服务器，导致 `/api/...` 返回 404。把后端地址填成 `http://localhost:8080` 后再点保存即可。浏览器里看到 `304` 不是报错，它只是告诉你静态资源命中了缓存。

超卖实验

这两个按钮都走真实数据库。朴素版会先 `select stock` 再 `update stock=?`，乐观锁版则会执行 `update ... where stock > 0`。

一人一单实验

同一个用户并发请求。先看不加锁时的重复下单，再看用户粒度锁如何把它挡住。

异步落库实验

这里主线程先受理请求并放入内存队列，后台线程再把订单写进数据库。示例里用队列模拟 Redis Stream / MQ，用 MySQL 模拟订单落库。
