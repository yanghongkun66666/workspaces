Java 秒杀演进实验室

这个页面把“超卖、乐观锁、一人一单、异步落库”都做成了可点击实验。你可以先在页面观察结果，再回到 Java 代码里对照 `SeckillDemoService` 一段段读。

超卖实验

先跑朴素版，再跑 `stock > 0` 版。通常你会看到前者 `successOrders` 大于 `initialStock`，后者不会。

一人一单实验

同一个用户并发请求。先看不加锁时的重复下单，再看用户粒度锁如何把它挡住。

异步落库实验

提交成功后会先进入“已受理”，随后后台消费者再把订单写入“已落库”。这对应秒杀里主链路快速返回、后台异步持久化的思路。

实验输出

重点关注 `oversold`、`duplicateOrder`、`acceptedOrders`、`persistedOrders` 这些字段。
